**Що це?**
Це спосіб захистити програму від помилок, які можуть зупинити її роботу.
*   **try**: Це "сховище". Тут ми пишемо код, який може зіпсуватися.
*   **catch**: Це "лікар". Якщо в `try` стається помилка, `catch` її ловить і лікує.

**Коли це треба?**
*   Коли ми не знаємо, чи введуть користувача правильні дані.
*   Коли програма може отримати помилку від системи (наприклад, файл не знайдено).
*   Коли потрібно зберегти програму від повного зупинення.

**Приклади коду**

```cpp
#include <iostream>
#include <stdexcept>

int main() {
    try {
        // Спробуємо поділити на нуль (це помилка!)
        int x = 10;
        int y = 0;
        int res = x / y; 
        std::cout << "Результат: " << res << std::endl;
        
    } catch (const std::runtime_error& e) {
        // Якщо сталася помилка часу виконання
        std::cout << "⚠️ Помилка: " << e.what() << std::endl;
    } catch (const std::exception& e) {
        // Якщо сталася будь-яка інша помилка
        std::cout << "⚠️ Загальна помилка: " << e.what() << std::endl;
    }

    std::cout << "Програма продовжує працювати!" << std::endl;
    return 0;
}
```

**Які винятки ми можемо ловити?**
*   `std::exception` — загальна помилка.
*   `std::runtime_error` — помилка під час роботи програми (наприклад, поділ на нуль).
*   `std::logic_error` — помилка в логіці програми (наприклад, неправильний аргумент).
*   `std::out_of_range` — ми зайшли за межі масиву або вектора.
*   `std::invalid_argument` — ми передали неправильний параметр у функцію.
*   `std::length_error` — ми намагаємося створити об'єкт, який занадто великий.
